Linear & Non-linear Systems

Solve linear and non-linear systems of equations. We first find roots of a single non-linear equation (fixed-point, bisection, Newton, secant), then solve linear systems by Gaussian elimination and matrix factorization (LU, Cholesky).

Root finding: the problem

A great many problems reduce to finding the root of an equation: a value \(x\) for which

\[ f(x) = 0. \]

Graphically, a root is where the curve \(y=f(x)\) crosses the \(x\)-axis. For linear or simple quadratic \(f\) we can solve by hand. But for most non-linear equations — \(\cos x = x\), \(x^3 - 2x - 1 = 0\) — there is no closed-form solution, so we approximate the root with an iterative method: start from a guess, improve it repeatedly, and stop when the change is below a tolerance \(\varepsilon\).

The four methods below trade off robustness (will it converge?) against speed (how fast?).

2.1 Fixed-point iteration

The idea: algebraically rearrange \(f(x)=0\) into the form \(x = g(x)\). A solution of this is a fixed point — a value the function maps to itself, \(g(x^\*)=x^\*\). We then iterate

\[ x_{k+1} = g(x_k), \qquad k = 0,1,2,\dots \]

and stop when \(|x_{k+1}-x_k| < \varepsilon\). The link to the original problem is \(f(x) = g(x) - x\): where \(g(x)=x\), we have \(f(x)=0\).

📐 Convergence condition
The iteration converges to a fixed point \(x^\*\) only if \(|g'(x^\*)| < 1\) near the root. Choosing a good rearrangement \(g\) is the whole art of the method — a poor choice diverges.

Worked example

Solve \(2x^3 - 2x - 5 = 0\). Rearrange to isolate \(x\):

\[ x = \left(\frac{2x+5}{2}\right)^{1/3} \;=\; g(x), \quad\text{which satisfies } |g'(x)|<1 \text{ near } x=1.5. \]

2.2 Bisection — the reliable bracket method

Bisection needs two starting guesses \(a\) and \(b\) that bracket the root: \(f(a)\) and \(f(b)\) must have opposite signs, so by the Intermediate Value Theorem a continuous \(f\) must cross zero between them.

  1. Compute the midpoint \(c = \dfrac{a+b}{2}\).
  2. Check the sign of \(f(c)\). Keep whichever half still contains the sign change: if \(f(a)\,f(c) < 0\) set \(b=c\), otherwise set \(a=c\).
  3. Repeat. The bracket halves every step, so the error is at most \(\dfrac{b-a}{2^{n}}\).
💡 Guaranteed but slow
Bisection always converges for a sign-changing continuous function, but only linearly — it gains roughly one bit of accuracy per step. It is the dependable fallback when faster methods misbehave.

2.3 Newton's method

Newton's (Newton–Raphson) method uses the slope of \(f\) to leap toward the root. From a guess \(x_n\), draw the tangent line to \(y=f(x)\); where that tangent crosses the \(x\)-axis is the next guess:

\[ x_{n+1} = x_n - \frac{f(x_n)}{f'(x_n)}. \]

It requires the derivative \(f'\) and a reasonable starting guess, but when those are available it converges quadratically — the number of correct digits roughly doubles each step.

Worked example

Solve \(\cos x - x = 0\) from \(x_0 = \pi/4 \approx 0.785398\), with \(f'(x) = -\sin x - 1\):

n\(x_n\)\(f(x_n)\)\(f'(x_n)\)\(x_{n+1}\)
00.785398−0.078291−1.7071070.739536
10.739536−0.000755−1.6730120.739085
20.739085−0.0000001−1.6730330.739085 ✓

Three steps reach the true root \(x \approx 0.739085\) to six decimals — far faster than bisection.

⚠️ When Newton fails
If \(f'(x_n)\) is near zero the step explodes; a poor starting guess can send the iteration to a different root or diverge. That is why robust solvers combine Newton with a bracket (a hybrid method).

2.4 The secant method

The secant method is Newton's method with the derivative replaced by a finite-difference estimate from the two most recent points. No derivative needed:

\[ x_{n+1} = x_n - f(x_n)\,\frac{x_n - x_{n-1}}{f(x_n) - f(x_{n-1})}. \]

It needs two starting points but no \(f'\), and converges superlinearly (order \(\approx 1.618\)) — slower than Newton, faster than bisection. A practical compromise when the derivative is awkward to compute.

2.6 Solving linear systems: Gaussian elimination

Now we turn from one equation to systems \(A\mathbf{x} = \mathbf{b}\). Gaussian elimination is the workhorse direct method. It uses three elementary row operations — swapping rows, scaling a row by a nonzero constant, and adding a multiple of one row to another — to transform the augmented matrix \([A\,|\,\mathbf{b}]\) into row-echelon (upper-triangular) form.

  1. Forward elimination: introduce zeros below each pivot, column by column, until the system is upper triangular.
  2. Back substitution: solve the last equation for the last unknown, then work upward.

Worked example

Find the parabola \(y = ax^2 + bx + c\) through \((3,25)\), \((1,5)\), \((-2,20)\). Substituting the points gives a \(3\times3\) system; elimination + back-substitution yields

\[ a = 3, \quad b = -2, \quad c = 4 \;\Rightarrow\; y = 3x^2 - 2x + 4. \]

2.7 LU factorization

LU decomposition factors a square matrix into a lower-triangular \(L\) (ones on the diagonal) times an upper-triangular \(U\):

\[ A = L\,U. \]

This is Gaussian elimination's bookkeeping made reusable. Once you have \(L\) and \(U\), solving \(A\mathbf{x}=\mathbf{b}\) is two cheap triangular solves:

\[ L\mathbf{y} = \mathbf{b}\ \text{(forward)},\qquad U\mathbf{x} = \mathbf{y}\ \text{(backward)}. \]
💡 Why factor?
The expensive elimination is done once. To solve the same \(A\) against many right-hand sides \(\mathbf{b}\), you reuse \(L\) and \(U\) — only the two fast triangular solves repeat. Hugely efficient for repeated solves.

2.8 Cholesky factorization

When \(A\) is symmetric and positive definite (all eigenvalues positive), it has a special, cheaper factorization into a lower-triangular matrix times its own transpose:

\[ A = L\,L^{\mathsf{T}}. \]

Cholesky is about twice as fast as LU and numerically very stable — the method of choice for the symmetric positive-definite systems that arise in least-squares (Module 3) and many engineering problems. Solving uses the same two-step idea:

\[ L\mathbf{y} = \mathbf{b}, \qquad L^{\mathsf{T}}\mathbf{x} = \mathbf{y}. \]

Worked example

Solve the symmetric system

\[ \begin{aligned} 6x + 15y + 55z &= 76\\ 15x + 55y + 225z &= 295\\ 55x + 225y + 979z &= 1259 \end{aligned} \]
